LLDB (Low Level Debugger) — это современный отладчик, разработанный как часть проекта LLVM. Он используется по умолчанию на macOS (вместе с Xcode) и доступен на Linux. LLDB имеет более современный интерфейс, чем GDB, и лучше интегрируется с Rust через rust-lldb. Он поддерживает те же базовые операции, что и GDB, но команды немного различаются. Для установки rust-lldb обычно достаточно установить компонент через rustup: rustup component add rust-lldb. После этого LLDB автоматически использует улучшенное форматирование для Rust-типов.
Чтобы запустить программу под LLDB:

    Соберите её в debug-профиле.
    Запустите LLDB с путём к бинарнику: cargo build и затем lldb target/debug/your_project.
    Или запустите LLDB и затем укажите файл через команду file target/debug/your_project.
    После запуска LLDB загрузит программу, но не начнёт выполнение. Чтобы запустить программу, используйте команду run (или r).
    Если программа принимает аргументы командной строки, передайте их после run, например: run arg1 arg2.
    Чтобы остановить выполнение программы, нажмите Ctrl+C — это прервёт выполнение и вернёт управление в LLDB.

Точки останова
Синтаксис LLDB немного отличается от GDB. Чтобы установить точку останова, используйте breakpoint set (или b):

# Остановка на функции
(lldb) breakpoint set --name main
(lldb) b main  # короткая форма

# Остановка на строке файла
(lldb) breakpoint set --file src/main.rs --line 10
(lldb) b src/main.rs:10  # короткая форма

# Остановка на адресе (редко используется)
(lldb) breakpoint set --address 0x400000 

После установки точки останова LLDB покажет номер breakpoint и место остановки. Чтобы посмотреть все установленные точки останова, используйте breakpoint list (или br list). Чтобы удалить точку останова, используйте breakpoint delete (или br del) с номером breakpoint. Чтобы временно отключить точку останова без удаления, используйте breakpoint disable и breakpoint enable для включения обратно.
Чтобы установить условную точку останова в LLDB, используйте breakpoint set --condition:

# Остановка, только когда переменная равна определённому значению
(lldb) breakpoint set --file src/main.rs --line 20 --condition "x == 42"
(lldb) b src/main.rs:20 -c "x == 42"  # короткая форма

# Остановка только после определённого количества итераций
(lldb) breakpoint set --file src/main.rs --line 15 --condition "i > 100" 

Также можно установить обычную точку останова и затем добавить условие через команду breakpoint modify. Чтобы посмотреть условие точки останова, используйте breakpoint list. Чтобы удалить условие, используйте breakpoint modify N --condition "" — это удалит условие для указанного breakpoint.
Пошаговое выполнение
Команды пошагового выполнения в LLDB аналогичны GDB: 

    step (или s) выполняет одну строку кода, заходя внутрь функций;
    next (или n) выполняет одну строку кода, не заходя внутрь функций;
    finish (или fin) выполняет оставшуюся часть текущей функции и останавливается на строке, где функция была вызвана;
    continue (или c) продолжает выполнение программы до следующей точки останова или до конца программы.

Синтаксис и поведение аналогичны GDB, поэтому, если вы знаете команды GDB, легко переключиться на LLDB.
Просмотр переменных и стека
Команды такие:

    print (или p) — чтобы посмотреть значение переменной. Вместе со значением вы увидите его тип. Для Rust-типов rust-lldb улучшает отображение аналогично rust-gdb.
    frame variable (или fr v) — чтобы посмотреть все локальные переменные текущей функции. Без аргументов эта команда покажет и локальные переменные, и аргументы функции.
    thread backtrace (или bt) — чтобы посмотреть стек вызовов.
    frame select (или f) с номером кадра — чтобы перейти к другому кадру стека.
    source list (или l) — чтобы посмотреть исходный код вокруг текущей строки.

Изменение переменных
Чтобы изменить значение переменной в LLDB, используйте expression (или e), например: expression x = 42 или e x = 42. После изменения переменной можно продолжить выполнение программы с новым значением.
